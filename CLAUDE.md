# CLAUDE.md

## プロジェクト概要

このフォルダは、理学療法士養成校に通う**4年生**が、**治療学臨床実習**のデータをまとめるための課題用フォルダです。

## 背景・目的

- 理学療法士（PT）を目指す学生が、臨床実習中に経験した症例や治療内容をデイリーノートとして記録・整理する
- 実習で学んだ内容（評価・治療・患者対応など）を振り返り、知識と技術の定着を図ることが目的

## 作業ルール

- Word（`.docx`）ファイルとMarkdown（`.md`）ファイルを自動で作成し、このフォルダ内に保存する
- ファイル名には実習日の日付を含める（例：`デイリーノート4月17日.docx`）
- 変更のたびに Git でコミットし、GitHub リポジトリ（`daily-note-Fourth-grader-`）へプッシュして記録を残す

## 文章作成スタイル

デイリーノートの文章は、「勉強は苦手だけど4年生まで進級してきた理学療法士の学生」が書いたようなイメージで作成する。
専門用語は適切に使う。一生懸命考えた結果として表現がやや素朴になるニュアンスは残しつつ、文章として不自然にならない形にまとめる。

### デイリーノートの記入項目（①〜⓾）

Wordファイル（`1_デイリーノート(治療学臨床実習).docx`）には以下の項目がある。
ユーザーが各項目の内容を伝えるので、それを整理してまとめ、**各項目名の右隣（同じ行）**に文章として記入する。

| 番号 | 項目名 |
|------|--------|
| ① | 見学内容 |
| ② | 情報収集 |
| ③ | 検査測定 |
| ④ | 動作観察 |
| ⑤ | 統合と解釈 |
| ⑥ | 問題点の抽出 |
| ⑦ | 目標設定 |
| ⑧ | プログラム立案 |
| ⑨ | プログラム実施 |
| ⓾ | 再評価 |

### 文章作成の具体的なルール

- ユーザーが話し言葉や箇条書きで内容を伝えたら、それをデイリーノートらしい文体に整える
- 難しすぎる表現は避け、学生らしい素直な言葉で書く
- 専門用語は使うが、多用しすぎない。使う場合も自然な流れに組み込む
- 「〜を見学した」「〜を確認した」「〜が難しかった」など、経験を素直に述べる形にする
- 各項目は簡潔にまとめる（長すぎない）
